## 🤔 ADP 工作流的统一设计

与传统的智能体开发平台不同，**腾讯云智能体开发平台（ADP）采用统一的工作流设计，无需区分聊天流和工作流**。这种设计理念大大简化了开发流程，让开发者能够更专注于业务逻辑的实现。

### 🎯 核心优势

**统一的工作流架构**
- ADP 将对话交互和业务处理融合在同一个工作流中
- 内置 Agent 全局监控节点，智能处理对话流程
- 既能支持灵活的多轮对话，又能确保精准的流程执行

**智能对话管理**
- 所有与工作流相关的对话都经过统一的 Agent 处理
- Agent 根据用户对话内容智能跳转到对应节点
- 全局识别退出意图和寒暄意图，提供自然的人机交互

### 🔧 技术特点

**灵活的对话交互**
- 支持多轮对话，系统自动分析信息完整性
- 智能生成反问澄清话术，引导用户提供必要信息
- 支持灵活的节点跳转，无需预设固定的对话路径

**强大的节点系统**
- **信息收集类节点**：参数提取、选项卡等，用于收集用户信息
- **信息处理类节点**：大模型、代码、工具等，用于处理业务逻辑
- **基础节点**：开始、结束、条件判断等，控制工作流流程

### 🌟 应用场景

ADP 工作流适用于各种复杂的业务场景：

**开发票场景**
- 收集开票信息（抬头、税号等）
- 通过大模型进行信息归一化
- 调用工具验证信息并生成发票

**案件办理场景**
- 收集案件关键信息
- 基于逻辑判断进行不同处理
- 录入最终办理结果

**患者信息查询场景**
- 结合患者身份信息查询病例
- 对病例信息进行摘要总结
- 基于病例信息进行智能问答
